# Product Profile: Example — TaskFlow

> This is an example profile to show you the format.
> Copy this file, rename it to `yourappname.md`, and fill it in.
> Claude will load it automatically when working inside your project.

---

## Settings
- **Audience:** 🟢 Non-technical — freelancers, small business owners
- **Domain:** 💼 Business / professional (task and project management)
- **Vibe:** Friendly & warm
- **Fun level:** 😊 Warm but professional — friendly and human, not trying to be funny

## Voice Notes
- Speak like a calm, organised colleague — not a cheerleader
- "We" is fine — TaskFlow is a collaborator, not just a tool
- Never use: workflow orchestration, asynchronous, kanban methodology, swimlane
- OK to use: tasks, projects, deadlines, assign, track, done

## Domain-Specific Jargon Swaps

| Avoid | Use instead |
|-------|-------------|
| Workflow orchestration | How your work flows |
| Asynchronous task | Task that doesn't need an immediate reply |
| Kanban | Board view |
| Swimlane | Group / section |
| Assignee | Who's responsible |
| Due date delta | Days until due |
| Backlog | Tasks waiting to be started |
| Sprint | Work period |
| Velocity | How much you're getting done |
| Dependency | This needs to be done first |

## Approved Example Rewrites

**Section headers:**
- "Task Management" → "Your Tasks"
- "Workflow Automation" → "Save time with automations"
- "Resource Allocation" → "Who's working on what"

**Empty states:**
- No tasks: "Nothing on your plate. Add your first task and get moving. → Add task"
- No projects: "No projects yet. Create one to start organising your work. → Create project"
- No team members: "Flying solo? Invite your team to assign tasks and track together. → Invite people"

**Loading states:**
- "Loading tasks..." → "Getting your tasks..."
- "Syncing..." → "Saving your changes..."

**Success messages:**
- "Task created." → "Task added — assign it or set a due date to keep things moving."
- "Saved." → "All saved."

**Error messages:**
- "Invalid date format." → "That date doesn't look right — try DD/MM/YYYY."
- "User not found." → "We couldn't find that email address. Double-check it and try again."

## Emotional Tone Per Moment
- Loading: calm and reassuring ("Getting your tasks...")
- Empty state: encouraging invitation ("Nothing here yet — let's fix that.")
- Error: matter-of-fact + helpful next step, never alarming
- Success: warm confirmation + nudge to next action
- Destructive confirm: serious and specific about what will be lost
